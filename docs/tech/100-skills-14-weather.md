# 100 个 Skills 第 14 期：weather - 零配置天气查询工具

今天介绍一个非常轻量但特别实用的 skill：**weather**。

它的特点非常直接：**不需要 API Key，不需要复杂配置，直接就能查实时天气和预报**。对于日常查询、自动化提醒、出行前检查都很方便。

| 项目 | 内容 |
|------|------|
| Skill 名称 | weather |
| 类型 | 天气 / 信息查询 |
| 依赖 | `curl` |
| 外部服务 | wttr.in / Open-Meteo |
| API Key | 不需要 |

## 核心能力

weather skill 主要提供两种能力来源：

1. **wttr.in**：适合快速、直观地查天气
2. **Open-Meteo**：适合结构化 JSON 查询和程序处理

### 典型能力

- 查询某个城市当前天气
- 查看简洁的一行天气摘要
- 拉取完整天气预报
- 获取适合程序处理的 JSON 天气数据
- 作为其他自动化流程的输入数据源

## 安装与使用

这个 skill 的好处是几乎不用安装额外依赖，只要本地有 `curl` 就能工作。

### 1. 快速查询当前天气

```bash
curl -s "wttr.in/Shanghai?format=3"
```

示例输出：

```bash
Shanghai: ☀️ +26°C
```

### 2. 使用紧凑格式输出更多信息

```bash
curl -s "wttr.in/Shanghai?format=%l:+%c+%t+%h+%w"
```

这类格式适合做脚本提醒，比如：
- 当前天气
- 温度
- 湿度
- 风速

### 3. 查询完整天气预报

```bash
curl -s "wttr.in/Shanghai?T"
```

适合需要看更完整天气趋势的时候使用。

### 4. 使用 Open-Meteo 获取 JSON

```bash
curl -s "https://api.open-meteo.com/v1/forecast?latitude=31.23&longitude=121.47&current_weather=true"
```

这个接口更适合：
- 自动化脚本
- 程序处理
- agent 工作流中的结构化输入

## 优缺点

### ✅ 优点

1. **零配置**：几乎开箱即用
2. **无需 API Key**：没有密钥管理成本
3. **适合脚本化**：命令简单，便于自动化
4. **适合 agent 使用**：既支持文本，也支持 JSON
5. **风险低**：无敏感权限、无复杂依赖

### ⚠️ 缺点

1. **依赖外部服务可用性**：网络异常时会受影响
2. **不是专业气象系统**：适合日常使用，不适合高精度业务场景
3. **城市名查询需要规范输入**：带空格时要做 URL 编码
